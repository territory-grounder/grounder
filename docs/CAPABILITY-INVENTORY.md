# Capability Inventory — the subsystem map Territory Grounder reproduces

**Status:** founding document · **Provenance tags:** `[F]` foundation (predecessor design TG inherits) · `[R]` product reframe (multi-user / de-solo) · `[O]` audit overlay (security + quality hardening). Source ids appear inline — `spec/00x`, `INV-NN`, `paradigm-rule N`.

Siblings: see **ARCHITECTURE.md** (how these subsystems compose on Go + Temporal + PostgreSQL/pgvector), **CONSTITUTION.md** (the inviolable mechanical safety core), **THREAT-MODEL.md** (the INV-01..22 bindings), and **the-map-is-not-the-territory.md** (the manifesto).

## How to read this document

Territory Grounder (TG) is a greenfield Go/Temporal/PostgreSQL reimplementation of a proven governed-autonomy SRE platform. This inventory is the *bill of materials*: every subsystem the predecessor ran, what it does, and TG's disposition toward it. It is the map from a battle-tested single-estate system to a distributable, self-hosted, **single-organization multi-user** product.

Each subsystem carries a **carry-forward disposition** [F]:

| Disposition | Meaning |
|---|---|
| **inherit-core** | The IP is load-bearing and ports essentially as-is (mechanism + intent). |
| **inherit-improved** | Ports, but the substrate change (Temporal/Postgres/adapters/multi-user) or the audit overlay materially strengthens or reshapes it. |
| **reduce-scope** | Keep the reusable pattern; demote the concrete/outward-facing/site-coupled parts to an optional module or drop them. |
| **drop** | Superseded by the runtime substrate migration; deleted from the build entirely (INV-17: a retired capability is *deleted*, never left dormant). |

Two cross-cutting reframes apply to **every** row and are not repeated each time:

- **`[R]` paradigm-rule 1** — multi-user, single-org by default: one deployment serves one organization with many operators, roles, and teams. There is no `tenant_id` and no cross-org row-level-security isolation; the correlation key is a bare `external_ref` (unique within the org's own trackers). Authority is checked against the acting user/role, and least-privilege credentials (per-source, per-agent) keep each adapter and agent to its granted capability — defense-in-depth, not tenancy.
- **`[R]` paradigm-rule 8** + **[O]** — the **inviolable mechanical safety core** (deterministic orchestrator owns the effect channel; model is untrusted and only proposes; reversibility dial with a non-configurable NEVER-auto floor; two-lane fail model advisory-open / remediation-closed; predict-before-acting fail-closed with mechanical post-hoc verdicts; gate on **structure**, not command-string blocklists) is invariant across all org policy and is re-founded behind typed, authenticated, action-bound interfaces per the audit overlay.

Terminology is identical across all TG docs: **ActionManifest**; the execution classes **DETERMINISTIC / FAST_AGENT / STANDARD_AGENT / DEEP_INVESTIGATION / HUMAN_LED**; the three autonomy bands **AUTO / AUTO_NOTICE / POLL_PAUSE**; the **module system** (adapters/ interfaces + modules/ implementations); **Temporal**; the **LiteLLM model-gateway**; the **RBAC roles / approver graph**.

Everything else in this inventory is inherited (core / improved / reduce-scope) — never the engine, always the design.

## Area 5 — Integrations (the module system)

Every integration surface — **ingest / tracker / notifier+approval / CMDB / actuation / model-provider / observability** — is a loadable/unloadable **module**, not hardcoded. `[R]` corrections (module system): `adapters/` holds the stable **interfaces**; `modules/` holds loadable **implementations** + a small reference-adapter set + an SDK. Default mechanism (ADR-0005, proposed): **out-of-process governed plugins** — each module a separate process/container over a stable protocol (gRPC / go-plugin; **MCP** for tool/actuation modules), signed, capability-scoped, RBAC-enabled. `[O]` INV-17: a disabled/unregistered module has **NO execution path** (kills the "dead OpenClaw path still executable" class).

## Area 6 — Data model

The Postgres schema. Two cross-cutting reframes dominate this area: **(a)** the schema is **single-org** — tables are org-global with **no `tenant_id` and no row-level-security org isolation**; the correlation key is a bare **`external_ref`** (unique within the org's own trackers), and the **DML-only runtime role** (never DDL) is the privilege boundary; **(b)** split the two stores the predecessor conflated — a short **immutable audit spine** (retained by archival/sealing under an **org/compliance TTL + legal-hold**) vs a **purgeable operational body** (**org TTL + hard-delete/right-to-erasure**). `[R]` rules 1/5, tension-resolved. `[O]` INV-16: **one Postgres reached through one DSN**, DDL only via ordered transactional migrations at deploy/startup (never in request handling), a DML-only runtime role, and integrity by construction (FK/NOT NULL/CHECK/enum).
