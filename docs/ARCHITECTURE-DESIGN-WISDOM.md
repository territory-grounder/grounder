# Architecture Design-Wisdom Map

## 1. How to read this

This is an **apples-to-apples internal-design and plumbing map**. For each concrete code object it asks a single question: *given what the sources (Gulli's* Agentic Design Patterns*, Anthropic's* Building Effective Agents *and* Writing Effective Tools*, the OpenAI Evals and Retrieval guides, Microsoft Semantic Kernel, LangChain context-engineering/middleware, and the canonical papers — ReAct, Reflexion, Constitutional AI, MCP, the async request-reply pattern — teach about how this **kind** of object should be built internally, how does TG's implementation of that object compare?* It is **not** a should-it-exist critique: every object here is taken as legitimate, and the only verdicts are `matches` / `exceeds` / `diverges` / `falls-short` on **internal design and wiring**, plus concrete, source-grounded improvements with an effort estimate (S/M/L). Section 2 walks the objects grouped by KIND; Section 3 — the load-bearing half — maps the cross-object plumbing (context flow, DI, return paths, observability); Section 4 is the prioritized backlog. There are no cut/merge/rename recommendations and no commentary on object counts anywhere.

## 2. Per-object design map

### Reasoning loop & tool interface

**`agent` — ReAct loop** (`agent/loop.go`, `confidence.go`, `trajectory.go`). *Sources:* ReAct interleaves Thought→Action→Observation and uses the explicit thought trace to track the plan and **handle exceptions**; Reflexion turns failure into re-injected verbal feedback; Gulli ch17 makes reasoning explicit for auditability and routes hard problems to a reasoning tier; Anthropic requires stopping conditions (iterations, tokens, time, confidence) and orchestration-intercept so no model token becomes control flow. *TG today:* a bounded typed-switch loop that calls only the LiteLLM `Completer`, first-JSON-decodes a `directive`, dispatches `tool|propose|stop|default→fail-closed`, treats confidence as first-class and fail-closed, vetoes thrash via `trajectory.go`, and runs three deterministic in-loop critics (citation gate, grounded-stop nudge, TG-60 decide-nudge that switches to the `DecisionModelName` tier for the one forced decision). *Alignment:* orchestration-intercept and no-token-as-control-flow **exceed** the sources (INV-08 by construction); dynamic model switching **exceeds** (scaling-inference paid once); deterministic Producer-Critic **matches**. It **diverges** on the explicit thought trace — the preamble mandates "EXACTLY ONE JSON object and NOTHING else," so the ReAct/CoT thought channel is actively suppressed; it **falls-short** on tool-error handling (a Go-error on a tool `OutcomeStop`s the whole session instead of feeding the error back as a recoverable ReAct exception) and on stopping-condition completeness (cycle limits and confidence but no token-budget cap and no per-cycle wall-clock deadline). *Improvements:* add an optional size-capped `thought` field to the directive parsed **only as data** (ReAct + ch17, **M**); on a tool error append an actionable `TOOL_ERROR[id]` observation and let the model try a different tool (ReAct exception-handling, **M**); add token-budget and context-deadline stops (Anthropic 6.4, **S**); route the reasoning tier up-front by `core/risk`/`core/execclass` band rather than only at the poll limit (ch17 dynamic switching, **S**).

**`agent tools` — the ACI** (`agent/tools.go`). *Sources:* Anthropic's Writing-Effective-Tools makes the Agent-Computer Interface a first-class contract — the model must know how to call a tool "from the description and PARAMETERS alone"; use poka-yoke typed/enum args, actionable errors, semantic ids over UUIDs, response-format verbosity control, and consolidation into semantic actions. *TG today:* the `Tool` interface exposes only `Name()`; args are an untyped `map[string]string`; the preamble surfaces a bare comma-joined name list. *Alignment:* semantic ids **match** (and are load-bearing — `ToolResult.ID` is the evidence anchor); reads-safe/writes-un-registerable **exceeds** SK's approval-gating; `get-estate-context` consolidation **exceeds**. It **falls-short** badly on the core ACI teaching: no `Description()`, no parameter schema/type/enum — the model must guess that `get-estate-context` needs a `host`, and the loop's `argMap` over-tolerance is a downstream symptom of that missing contract. *Improvements:* add `Description()` + a typed param schema (name/type/required/enum/example) rendered into the preamble (Anthropic ACI, **M**) — the single highest-leverage reasoning-layer fix; layer enum/required validation over the arg map returning an actionable message (poka-yoke, **M**); surface `ToolResult.Success` into the observation so model and judge see explicit tool outcomes (**S**).

### Retrieval & memory

**`core/knowledge` — retriever** already **matches or exceeds** most of the RAG wisdom: a real pgvector cosine channel fused with a transparent lexical scorer by RRF (Gulli/OpenAI hybrid), citations-with-reasons that **exceed** the source-attribution teaching, and an injection defense (data-framing + `core/screen` on every snippet) that **exceeds** LangChain 5.8. It **diverges** where the `MinSim=0.5` floor guards only the semantic channel (a sub-0.1 lexical summary-overlap still enters), where results are plain bullets rather than XML-boundaried blocks, and it does **not address** query expansion or recency ranking. *Improvements:* a lexical/fused relevance floor mirroring `MinSim` (OpenAI 5.5, **S**); XML per-source envelopes (OpenAI 5.6, **S**); a deterministic synonym-expansion table preserving INV-08 (OpenAI 5.3, **M**).

**Memory stores.** `core/lessons` **matches** LangChain's collection-not-profile + background-task write and **exceeds** on its quality gate (only match+ConfirmedClear becomes precedent), but **falls-short** on provenance — the `knowledge.Incident` row carries no resolved-at timestamp for the retriever to rank on (stamp it, Gulli Agentic-RAG, **S**). `core/skillstore` is the strongest object against its wisdom: it **exceeds** Gulli ch8 procedural-memory by validating self-modification through generate-only → offline gate → Welch A/B → safety-guard → auto-rollback, where the sources merely `store.put`. It **diverges** only in that `GenerateCandidates` passes the *dimension name* rather than Gulli's prescribed "recent interactions" — feed it a bounded screened set of low-scoring session excerpts (**M**). `core/learn` **matches/exceeds** the GraphRAG-substrate teaching but **falls-short** on reconciliation — co-occurrence counts accrue forever with no decay (add a half-life, Gulli ch14, **M**). `agent/skills` **exceeds** on conditional composition by exec-class but **diverges** on block boundaries (guidance shares one user message with untrusted data separated only by textual framing — the delimiter fix below).

### Prediction / verify / falsify

**`core/predict`** **exceeds** the external-verifier wisdom (a fully deterministic infragraph prediction, no LLM in the path) and bakes a degree-preserving negative control into **every** row (OpenAI negative-controls, INV-22). It **diverges** on OpenAI's "smooth scores, not pass/fail": `PredictedHosts` is flattened to a set, discarding the per-host path-product confidence the estate graph already computed — carry it through so verify/falsify can compute a Brier-style calibration score (**M**). **`core/verify`** **exceeds** with a *pure-function* verdict author (stronger than an LLM critic) but **falls-short** on structured rationale — it returns only the 3-way enum, discarding the surprise hosts and `(host,rule)` mismatches that the interceptor and falsify then recompute separately; return a typed `VerdictDetail` in one pass (Gulli ch4/ch19, **M**). **`core/falsify`** finally gives the predict→verdict→score lane a production caller and **matches** the continuous-external-measurement wisdom, but **falls-short** on the flywheel: a scored deviation only increments a counter — emit it into a discovery-set sink so real failures drive eval coverage (OpenAI 3.7, **M**), and use a per-alert-class observation window rather than a global 10-minute constant (Gulli ch11, **M**).

### Evaluators

`core/judge` **matches** the rubric/defensive-parse/separate-critic wisdom and is the single source of judge semantics, but **falls-short** on per-dimension rationale (one blended comment vs ch19's `detailed_feedback`) and does **not address** temperature pinning or few-shot calibration exemplars — declare a shared `JudgeParams{model, temperature:0, seed}` every caller must honor (OpenAI 3.4, **S**) and embed 2-3 scored exemplars (OpenAI 3.5, **M**). `core/governance` **exceeds** with an independent-**model** frontier cross-check (the external-of-the-external verifier that caught the 3-week dead-judge class) but **falls-short** on KPI time-series — persist each agreement-rate/judged-fraction reading as a leading indicator (ch19, **M**). `tools/evalgate` **exceeds** on same-window A/B drift-cancellation but **diverges** from the three-set model (no rolling discovery corpus) and **falls-short** on reproducibility (`VerifyComparable` checks corpus size, not that both arms used the same judge model/temperature — the drift-cancellation *assumes* it). `tools/rejudge` and `tools/shadowbench` share the same temperature-pin gap; shadowbench additionally **diverges** by reimplementing the rubric in `judge.py`, a second copy outside `core/judge`. The unifying fix — one judge model+temperature pin and one rubric source across all five evaluator surfaces — is the load-bearing evaluator improvement.

### Policy evaluator (`core/policy`)

The PDP (`engine.go`) is textbook: a Go match against the shared `credential.Selector/Target` grammar plus a fixed embedded Rego deny-overrides module (operators supply **data only**, INV-06) — it **exceeds** "decision is a structured object" and **matches** PDP/PEP separation. It **falls-short** on naming every rule that fired (`pickRuleID` collapses the computed `matched_ids` list to one id) and on carrying a policy **version** into every decision/ledger row; it **diverges** on fail-closed **direction** — both no-match and Rego eval-error resolve to `approve` (route-to-human), but an evaluator *error* should fail to `deny`. The composition pipeline (`band.go`) **matches** confidence/rate stopping-conditions and most-restrictive-wins and **exceeds** with the inviolable `neverAuto` clamp, but **falls-short** because the `RateGovernor` window is in-memory only (a restart resets the auto budget — back it with the graduation store's durable seam, LangChain 7.2, **M**). The graduation ladder (`graduation.go`) **exceeds** by binding promotion to *verified* outcomes and refusing an un-persisted promotion. The mode controller (`mode.go`) **exceeds** on audited fail-closed transitions and the `pipeline_guard` proof that verdict/band are byte-identical across all four modes. *Improvements:* thread a bundle version through `PolicyDecision` (OPA, **M**); surface the full matched-rule list (Gulli ch18, **S**); split fail-closed direction to `deny` on error (OPA, **S**); collect the scattered obligations (`approve_by`, `DoubleWarn`, `FloorClamped`) into one `[]Obligation` on the decision (XACML, **M**).

### Credential resolver (`core/credential`, `modules/credsource`)

The engine **matches** MCP's "one standard interface wrapping heterogeneous backends" and **exceeds** on authN/authZ-first-class (it composes after policy over the same object grammar) and the two-plane machine/human router. It **falls-short** on MCP discoverability — there is no `Describe`/capability manifest, `MembershipSource` is found by an ad-hoc type assertion, and no operational telemetry is emitted (durable audit rows exist, but no resolution-outcome counters or per-source sync latency/drift metrics). Source failures reach the caller as an opaque string, not a class. Among the backends, `vault` **matches** the safe-cache-with-invalidation teaching (403→relogin) but `awx`/`semaphore` **diverge** — they cache the bearer token once and never invalidate, so a rotated upstream token fails closed until restart. *Improvements:* add a `HealthProber`/`Describe` capability the `SyncEngine` can enumerate (MCP discoverability, **M**); emit resolve/sync telemetry via `core/metrics` (SK 8.1, **M**); make awx/semaphore caches re-resolve on 401/403 (SK 7.3, **S**).

### Guardrail chain

`core/screen` **exceeds** by screening **both** trust boundaries (inbound scrub-and-flag, outbound forced POLL_PAUSE) and **matches** CAI non-evasive handling, but **falls-short** on PII/secret redaction — it detects injection but does not redact a leaked credential embedded in an alert body before forwarding (add a secret-regex pass to `Scrub`, SK 6.3, **M**; a live NetBox token was in fact found in a predecessor plaintext log). The never-auto floor (`core/safety`) **exceeds** by re-deriving destructiveness from the *actual* command (defeating a lying model) and by immutability-of-the-constitution, but **falls-short** on auditability — the backstop regexes are compiled string literals, not an externalized reviewable data file (externalize + hash-lock, CAI, **M**); have `IsNeverAuto` return the matched slug so refusals explain themselves (**S**). The mode chokepoint **matches** fail-closed-default and kill-switch but **falls-short** on chokepoint-level observability (store a timestamped `lastKill{reason,at}`, SK 8.1, **S**). The interceptor (`core/actuate`) **exceeds** SK's before/after filter model with ~9 pre-gates + ~6 post-steps and fail-loud recording, but **falls-short** on transient-vs-permanent error classification (any `Exec` error is a flat refuse) — classify it and give refusals a typed `RefusalCode` enum (Gulli ch12/ch18, **M**). The breaker **exceeds** with lane-aware controlled failure (advisory fails open, safety fails to Shadow) but **falls-short** on two plumbing points: the safety `MutationBreaker` is wired with `NewMemStore()` so a trip in one worker does not force-Shadow siblings (CONSTITUTION:130 wants persisted cross-process state), and a trip pages nobody — it writes only the ledger (fan a trip to a notifier, Gulli ch12, **S**).

### Ingest / suppression / estate / risk

`core/ingest` **exceeds** on the typed inter-stage contract (`IncidentEnvelope`, no raw bytes flow downstream) but **falls-short** on Gulli ch1's iterative recovery — a malformed field is a one-shot hard reject with no deterministic re-derivation pass (host from IP, severity from rule) (**M**). `core/suppression` **exceeds** the rule-router wisdom (fail-open-to-escalate by construction, novelty defaults to safe) but **falls-short** on routing output — on escalate it emits a bare pass-through, no near-miss evidence for the downstream agent (emit a structured routing hint, Gulli ch2, **S**). `core/estate` **exceeds** on the max-confidence reconciliation ratchet and background atomic refresh but **falls-short** on hybrid/alias resolution — a host referenced under an alias or IP lands on a disconnected twin (add a fuzzy/alias tier, OpenAI 5.1, **M**) and confidence only ever ratchets **up** with no decay-on-disproof from verify/falsify (**L**). `core/risk` **matches/exceeds** the deterministic classify→band router but **falls-short** on transparency — the most-restrictive-first early return records only the *first* tripped gate, hiding that a POLL_PAUSE was also stateful+novel (record all tripped gates + a routing margin, Gulli ch1/ch2, **S**).

### Orchestrator & actuators

`RunnerWorkflow` **matches** the sequential-decomposition wisdom and **exceeds** on typed inter-step payloads with a content-hashed `action_id` threaded end-to-end, but **falls-short** on retry discipline — the base activity context sets no `RetryPolicy`, so a deterministic Classify/Gate logic error inherits Temporal's default unbounded retry (set a bounded policy + `NonRetryableErrorTypes`, Gulli ch12, **S**) — and has no workflow-level token/time budget stop (**M**). The composition root (`cmd/worker/main.go`) is a textbook SK kernel-centric DI seam and **exceeds** on the explicit fail-closed/fail-open split per injected dep, but **falls-short** on observable-by-default: it wires metrics only for the safety plane and never injects an emitter into the activities. `escalation.Controller` and `reconcile.Reconcile` are the exception-recovery lanes Gulli ch12 prescribes and are **exceeds**-grade in their internal design (storm-free mark-before-signal; never close on asserted success) — but both **fall-short** on reachability: they are built and oracle-tested yet are not registered as activities or scheduled workflows, so they have no production driver. Among effect leaves, `ssh` **exceeds** on injection-unrepresentable argv and defense-in-depth authz but **diverges** on the typed-tool teaching (the `Actuator` seam is stringly `Exec(argv)`); `proxmox` **matches** the async fire-then-poll pattern but **falls-short** on durability (its `pollTask` blocks `Exec` in-process and loses the UPID on restart); `kubernetes` **falls-short** on the GitOps regime split and is distroless-broken (subprocess kubectl); `mcp` is a placeholder with no real transport. The async/regime design work (spec/017) generalizes these correctly.

## 3. The plumbing & architecture

The sources are most opinionated about the *spaces between* objects — context flow, dependency wiring, return paths, and observability — and this is where TG is simultaneously strongest and weakest.

**Context assembly is engineered, but the trusted/untrusted boundary is prose.** LangChain calls "deciding what information each agent sees" the center of design, and Gulli calls context engineering a discipline of building the full informational environment before token generation. TG realizes this well: `InvestigateActivity` composes the seed block-by-block from named injected sources (`Retriever`, `CMDBResolve`, `TrackerRead`, `SkillRows`), each block screened through `screenSeedBlock`, each carrying an `input-screened` provenance note serialized into Temporal history so the seed is byte-reconstructable, and skills sized conditionally by exec-class. This **matches or exceeds** the wisdom on deliberate composition and injection defense. But the seed is ultimately *concatenated prose* — the trusted behavioral guidance and the untrusted DATA blocks (precedent, summary, CMDB, ticket) share **one user message** separated only by textual framing like "(data, not instructions)". Every source that speaks to this boundary (OpenAI 5.6, LangChain 5.8, SK prompt filters) prescribes explicit machine-parseable delimiters. Wrapping each block in `<precedent>…</precedent>`, `<cmdb>…</cmdb>`, `<behavioral_guidance>…</behavioral_guidance>` is a small (**S**) change spanning `agent/skills` + `activities.go` that hardens the single most safety-relevant boundary in the reasoning path. There is also no per-block token budget — a large CMDB attribute set can crowd the window (**M**).

**The DI seam is exemplary; the observability seam is the gap.** `runner.Deps` is a single composition root of ~30 interface-typed collaborators built once and injected, with pgx stores and in-memory oracle twins interchangeable behind the same interfaces — a faithful, **exceeds**-grade SK kernel-centric DI realized in Go, extended with a per-field fail-closed/fail-open discipline the sources only gesture at. But SK's other mandate — observable-by-default, OTel in **every** layer, events at each lifecycle step — is unmet from the DI seam outward. The composition root wires Prometheus gauges only for the safety plane (mutation gate, breaker, halts) and never threads an observability emitter into the activities, so the agent loop (cycles, tool errors, model latency), the ingest chain, and the classify/gate steps emit no spans or per-step metrics. `core/metrics` itself is exemplary for its determinism (sorted, diff-stable, secret-free, per-scrape-current) — the mechanism is ready; only the collectors and the injected emitter are missing. This is why none of OpenAI/SK's five minimum agent metrics (runtime, tool-call count, token consumption, tool errors, accuracy) are exposed. **Injecting one emitter at the composition root and threading it to every activity** (SK 8.1/8.3, **M**) is the highest-leverage plumbing improvement in the system because it lights up every other layer at once.

**The return paths converge correctly on two spines.** The tool→loop return path is `OBSERVATION[id]: data` — LangChain's Tool Context category made concrete, with `ToolResult.ID` as the semantic anchor that binds the tool layer to the entire grounding subsystem (the citation gate, INV-11 evidence-binding, `core/falsify`, `core/judge` all cite the *same* id). The decision return path converges on `core/audit`: ~12 writers funnel through one hash-chained, required-field, single-writer `Ledger.Append` — an **exceeds**-grade realization of SK's audit-logging-as-filter and Gulli's structured-required-record, extended with tamper-resistance (no UPDATE/DELETE role) the sources do not even describe. Both spines are load-bearing and well-built. The gaps are outbound: the ledger — the richest governance signal in the system — feeds **no metrics** (no decisions-total, no chain-broken gauge) and carries no trace/correlation identity beyond `action_id`, so a decision that *precedes* an action (suppression, halt) cannot be joined to the observability plane (LangSmith 8.2, **M**).

**The recovery half of the loop is built but not plumbed.** Gulli ch12 makes escalation, notification, and confirmed-clear close-out first-class recovery mechanisms that "must actually run when the failure occurs." TG has both lanes (`escalation.Controller`, `reconcile.Reconcile`) as complete oracle-tested primitives — `reconcile` even **exceeds** the wisdom by holding an executed remediation until its blast-radius verdict lands rather than trusting asserted success — but neither is in `RunnerWorkflow` nor registered on the worker. The workflow handles an unanswered poll inline (24h timer → deny) but never delegates to the re-check lane, and never drives a ticket close-out. Three wires are absent and named by the design: a terminal `RunnerResult → reconcile activity → tracker transition`; `reconcile.ScheduleReCheck → escalation.ScheduleReCheck`; and a scheduled `FireDue` workflow. Closing them (**L**) is what makes the exception-handling loop reachable.

**Shared-state discipline is strong except at the breaker.** The estate graph, suppression registry, and knowledge corpus all sit behind atomic-pointer holders (background-task memory writing, no hot-path latency) — **matches/exceeds**. The one contradiction: the safety `MutationBreaker` is wired with an in-process `MemStore` while CONSTITUTION:130 and the breaker's own doctrine require *persisted, cross-process* state. A deviation trip in one worker will not force-Shadow its siblings — the shared-kill guarantee the canary plan depends on is not yet global. Swapping to the pgx breaker store (**M**) is a prerequisite before any multi-worker canary, and must be paired with a shared posture store so the kill is truly system-wide.

## 4. Design-improvement backlog (ordered by leverage)

1. **Inject an observability emitter at the composition root and thread it into every activity**, then add the five-metric family and ledger-derived metrics to `core/metrics`. *Source:* SK observable-by-default (8.1) + five eval metrics (8.3). *Object:* `cmd/worker/main.go` → `runner.Deps` → `core/metrics` + `core/audit`. *Effort:* M/L. *Leverage:* lights up the reasoning, ingest, and ledger layers that are currently observability-dark.

2. **Wire `reconcile` and `escalation` into the Temporal worker** — a terminal reconcile activity driving ticket close-out, a scheduled `FireDue`, and the `reconcile→escalation` re-check hand-off. *Source:* Gulli ch12 (recovery must be reachable). *Object:* `core/reconcile`, `core/escalation`, `temporal/runner/workflow.go`. *Effort:* L.

3. **Back the safety `MutationBreaker` with the pgx cross-process store (plus shared posture)** so a deviation trip force-Shadows every sibling worker. *Source:* CONSTITUTION:130 + Gulli ch12 (persisted cross-process state). *Object:* `core/safety/mutation_breaker.go` wiring in `cmd/worker/main.go`. *Effort:* M.

4. **Wrap each seed block in explicit machine-parseable delimiters** separating trusted guidance from untrusted data. *Source:* OpenAI Retrieval 5.6 + LangChain 5.8. *Object:* `agent/skills` + `temporal/runner/activities.go`. *Effort:* S. *Leverage:* hardens the highest-risk boundary in the reasoning path at low cost.

5. **Add `Description()` + a typed parameter schema to the `Tool` interface** and render both into the preamble. *Source:* Anthropic Writing Effective Tools (ACI). *Object:* `agent/tools.go`. *Effort:* M. *Leverage:* removes the root cause the loop's `argMap` over-tolerance treats as a symptom.

6. **Turn a tool Go-error into a recoverable `TOOL_ERROR[id]` observation** instead of aborting the session. *Source:* ReAct exception-handling + Gulli ch5. *Object:* `agent/loop.go`. *Effort:* M.

7. **Restore an optional, size-capped `thought` field to the directive** (parsed only as data, INV-08 intact). *Source:* ReAct + Gulli ch17. *Object:* `agent/loop.go`. *Effort:* M.

8. **Set a bounded activity `RetryPolicy` + a workflow token/time budget stop.** *Source:* Gulli ch12 + Anthropic 6.4. *Object:* `temporal/runner/workflow.go`. *Effort:* S/M.

9. **Unify the judge across all five evaluator surfaces** — one `JudgeParams{model,temperature:0,seed}` every caller honors, and one rubric source (`core/judge`) that `eval/gate` and `shadowbench/judge.py` read rather than re-declare. *Source:* OpenAI Evals 3.4 + one-judge principle. *Object:* `core/judge`, `tools/evalgate`, `tools/rejudge`, `tools/shadowbench`, `eval/`. *Effort:* M.

10. **Feed scored deviations from `core/falsify` into a rolling discovery corpus** that promotes into the regression suite. *Source:* OpenAI Evals 3.5/3.7 (three-set flywheel). *Object:* `core/falsify` → `eval/` + `tools/evalgate`. *Effort:* M/L.

11. **Add a shared recency/decay discipline across the three learned stores** — provenance timestamps on lessons, half-life on `core/learn` counts, decay-on-disproof on estate edges. *Source:* Gulli ch14 (periodic reconciliation). *Object:* `core/lessons`, `core/learn`, `core/estate`. *Effort:* M.

12. **Harden the policy decision surface** — bundle version threaded into every `PolicyDecision`/ledger row, full matched-rule list surfaced, and fail-to-`deny` on evaluator error. *Source:* Web OPA best practice + Gulli ch18. *Object:* `core/policy/engine.go`. *Effort:* S–M.

13. **Add a secret/PII redaction pass to `Scrub`** so a leaked token in an alert body is redacted before it reaches the model or logs. *Source:* SK 6.3 (prompt-filter PII removal). *Object:* `core/screen/screen.go`. *Effort:* M.

14. **Move `proxmox` task-polling out of `Exec` into the deferred-verify channel** with a durable pending-verification record keyed by `action_id`. *Source:* Azure async request-reply + spec/017. *Object:* `modules/actuation/proxmox`, `core/actuate` verify tail. *Effort:* L.

15. **Return a typed `VerdictDetail` from `core/verify` in one pass** (surprise hosts + mismatches) instead of the bare enum the interceptor and falsify recompute. *Source:* Gulli ch19 rubric + ch4 structured critic. *Object:* `core/verify`. *Effort:* M.
