# NVIDIA DLI — RAG / Agents / Data Flywheel

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Course** · TG adherence: **3.2 / 5** · Practices audited: **21** (implemented 11 · partial 4 · gap 5 · N/A-by-design 1).

## Verdict

TG is a native Go triage agent, not a RAG chatbot, so it inverts this source's emphasis: it is excellent exactly where the DLI agentic course is strongest and weak exactly where the DLI RAG deck is strongest. The data-flywheel (aggregate→evaluate→customize→version→observe) is fully wired and arguably EXCEEDS the predecessor — the skill/prompt graduation loop with a Welch t-test and post-graduation auto-rollback is real code, where the predecessor completed 0 trials. Guardrails and chat-fragility are strong: a fail-closed mechanical safety core, a 5-category prompt-injection screen with homoglyph/zero-width folding, PII redaction on egress, and a hard citation + grounded-stop evidence gate that beats the source's check_evidence pattern. The one material regression is retrieval — TG deliberately dropped to transparent lexical-only precedent matching, so the RAG deck's hybrid + cross-encoder rerank + RAG-Fusion + HyDE + asymmetric-embedding stack (all of which the predecessor HAD) is absent, and pgvector is provisioned in deploy but touched by no Go code. Secondary gaps: the injection screen runs on the proposal, not on the untrusted ingest input, and there are no NeMo-style semantic topic rails on intermediate steps.
