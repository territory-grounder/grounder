# OpenAI — Retrieval Guide

> **[O] audit-overlay · TG-38 source benchmark.** Category: **OpenAI** · TG adherence: **2.1 / 5** · Practices audited: **7** (implemented 1 · partial 3 · gap 2 · N/A-by-design 1).

## Verdict

TG's retrieval plane as actually built is a single transparent lexical retriever (exact rule/host/site + tag/summary token overlap) over an in-memory JSON incident corpus — NOT the 5-signal RRF / pgvector / rerank / query-rewrite pipeline its own docs mark "inherit-core." SYSTEM-MAP.md:115 honestly concedes that plane is unbuilt (no vector column — confirmed absent from every migration — no embeddings, no RRF, no rerank). Consequently TG meets the OpenAI guide strongly only on the safety-critical row, RAG security, which it arguably exceeds: retrieved context is explicitly framed as data-not-instructions, INV-08 stops any model token becoming control flow, and a deterministic citation gate (REQ-1007) plus grounded-stop (REQ-1008) validate every proposal against actually-captured evidence. Attribute filtering, score thresholding, and XML result wrapping are present only in spirit (exact-match ranking, a trivial score>0 floor, plain-text delimited blocks); hybrid/RRF search and query rewriting are absent from code; chunking is N/A because the retrieval unit is an atomic incident record, not a long document.

## Top gaps

- Wire the pgvector semantic channel + RRF hybrid fusion (retrieval is keyword-only today)
- Add a configurable minimum-relevance score threshold to the retriever (current floor is a trivial score>0)
- Add query expansion/rewrite for the free-text summary channel
- Wrap precedent/CMDB/ticket DATA blocks in explicit XML delimiters for boundaries + attribution
