# LangChain — LangGraph Platform GA

> **[O] audit-overlay · TG-38 source benchmark.** Category: **LangChain** · TG adherence: **4 / 5** · Practices audited: **5** (implemented 3 · partial 2 · gap 0 · N/A-by-design 0).

## Verdict

TG's Temporal substrate meets or exceeds LangGraph's durability and human-in-the-loop story: the Runner is a deterministic event-sourced workflow where every side effect is an activity, human approval is a durable 24h signal-wait with deny-by-default and action-id binding (INV-12) that is stricter than a vanilla LangGraph interrupt, and Temporal's replayable event history is a first-class checkpointer with GetVersion migration guards. The honest shortfalls are the orchestrator-worker pattern (a linear activity pipeline with a declared-but-unbuilt fast/deep execclass split, no dynamic child-workflow fan-out/synthesize) and continue-as-new (adopted in ADR-0002 but never called in code; long-running history is bounded only implicitly by the CronSchedule option, and the per-incident Runner is short-lived so it is largely N/A). A secondary durability nuance: the agent ReAct loop runs as one atomic activity, so intermediate reasoning cycles are not individually checkpointed the way LangGraph checkpoints each node.

## Top gaps

- Build execclass-routed fast/deep orchestrator-worker paths with child-workflow fan-out for correlated incidents
- Checkpoint the agent ReAct loop at cycle granularity so a mid-investigation failure resumes instead of restarting
- Reconcile continue-as-new: make it explicit for long-lived workflows or document CronSchedule/short-lived-runner coverage
