# Microsoft — Agent Framework 1.0

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Microsoft** · TG adherence: **3.9 / 5** · Practices audited: **5** (implemented 2 · partial 3 · gap 0 · N/A-by-design 0).

## Verdict

TG's adherence to MS Agent Framework 1.0 is strong exactly where the framework's own architecture criterion points: 'if you can write a function, do that instead of an agent' is not just met but structurally enforced (INV-08 — the LLM never becomes control flow; every gate, classifier, verdict-writer, and retriever is a pure Go function), and checkpointing is exceeded via Temporal durable execution (durable timers, replay-versioned workflows, byte-reconstructable history). Context providers for memory are genuinely wired (skills + RAG precedent + CMDB + tracker, refreshable via a lessons feed) but shallow — lexical retrieval over a fixed corpus rather than the pgvector-backed semantic memory the stack advertises. Multi-agent orchestration is partial and largely deliberate for a single-org triage system: a real generator-evaluator flywheel exists across durable Temporal roles, but there is no concurrent/handoff/group-chat collaboration and the 'sub-agents-as-tools' pattern the README claims is unimplemented. Streaming is the weakest area — no model-token or in-flight session streaming (an SSE posture stream exists, but you cannot watch a triage session unfold).

## Top gaps

- Stream in-flight triage sessions to the console (SSE of cycles/observations/proposal), not just aggregate posture
- Wire pgvector/embedding-backed semantic precedent retrieval behind the knowledge.Retriever interface
- Resolve the sub-agents-as-tools doc/impl mismatch in agent/README.md — implement the manager pattern or mark it deferred
