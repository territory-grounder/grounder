# Google — Choose Agentic AI Architecture Components

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Google** · TG adherence: **3.1 / 5** · Practices audited: **10** (implemented 5 · partial 2 · gap 2 · N/A-by-design 1).

## Verdict

TG's strongest match to this source is runtime selection: it deliberately adopts Temporal durable execution (durable timers, signals, versioned replay, at-most-once mutate) plus a bundled LiteLLM model-runtime and loadable modules — a clear, well-argued runtime choice that beats the predecessor's n8n/Cronicle sprawl. Its memory story is broad and unusually well-governed (working/trajectory, hash-chained episodic audit spine, a governed observe→resolve→learn→retrieve corpus, and a procedural skill-store graduation flywheel) but is missing the one thing ADR-0003 and this source most emphasize: semantic/vector (embedding) memory — retrieval is lexical-only and no migration ever creates a vector column. On model routing by task complexity TG only partially delivers: it splits by CALLER TYPE (a fast Mistral loop model vs a Kimi-K3 primary reasoner for judge/synthesis), which honors the spirit, but there is no per-incident complexity classifier driving model choice and no adjustable thinking budget (the exact predecessor gap G11). Design-pattern selection exists as a real typed execution-class selector (execclass) and rich concrete patterns (ReAct, reflection-judge, RAG, HITL), but the selected class currently only steers skill composition — it does not yet branch the model tier, deep-vs-fast context, or the deterministic no-agent shortcut.

## Top gaps

- Wire execclass to drive model tier + deep/fast context (per-incident complexity routing) instead of the fixed 'fast' tier — closes predecessor gap G11
- Deliver ADR-0003's pgvector semantic/vector memory (embedding column + embedder + semantic Retriever) — retrieval is lexical-only today
- Add an adjustable per-task thinking/reasoning budget (reasoning_effort or token tiers) keyed off exec class
- Honor the Deterministic (no-agent) and FastAgent (compact context) topologies so pattern selection actually changes runtime behavior, not just skill composition
