# Google — Agent2Agent (A2A) Protocol

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Google** · TG adherence: **2.8 / 5** · Practices audited: **8** (implemented 3 · partial 2 · gap 2 · N/A-by-design 1).

## Verdict

A2A is fundamentally an inter-agent interoperability wire protocol (Agent Cards, JSON-RPC envelopes, discovery, task lifecycle). TG deliberately runs ONE native Go ReAct loop in a single org, so the federation surface is legitimately deferred — and the predecessor's own A2A bus was live-proven dead (a2a_task_log = 53 completion-only rows abandoned after 2 weeks), which vindicates the de-scope. Where TG is strong is the SUBSTANCE the task-lifecycle practice protects: a content-hashed ActionManifest stage chain, a hash-chained audit ledger (state-transition history that exceeds A2A's optional capability), and an authenticated human-in-the-loop escalation/vote path (the input-required analog). Where TG is weak is the interoperability ARTIFACTS: there is no Agent Card, no well-known discovery path, no JSON-RPC/typed handoff envelope anywhere in code. The sharpest finding is an honesty gap: CONSTITUTION §4.14 claims "the capability-card / versioned-envelope inter-agent contract is kept," but no such artifact exists — SYSTEM-MAP itself flags this and recommends either building the minimal contract or downgrading the claim.

## Top gaps

- Publish a minimal self-describing AgentCard (or downgrade CONSTITUTION §4.14's 'capability-card contract is kept' claim to the single-loop reality)
- Define a versioned handoff/delegation envelope, or explicitly scope out the A2A wire envelope — today there is no JSON-RPC/typed inter-agent message format
- Build DEEP_INVESTIGATION read-only sub-agent delegation binding, or downgrade the constitution's claimed-but-unbuilt manager-pattern routing
