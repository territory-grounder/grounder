# OpenAI — A Practical Guide to Building Agents

> **[O] audit-overlay · TG-38 source benchmark.** Category: **OpenAI** · TG adherence: **4.5 / 5** · Practices audited: **12** (implemented 10 · partial 1 · gap 0 · N/A-by-design 1).

## Verdict

TG is a strong, safety-forward realization of this guide and in several dimensions exceeds it. It nails the guide's central principle — use the LLM for ambiguous judgment (open-ended read-only triage) while ALL control flow and safety stay deterministic ("no model token becomes control flow", INV-08) — and its guardrail story is deeper than the source recommends: layered defense-in-depth (injection screen → deterministic risk bands → prediction gate → wired interceptor chain → mechanical verifier → auto-trip breaker), a strict single-grammar + citation output gate, per-tool risk-rating with human-approval on high-risk bands, failure/retry thresholds that escalate, and a genuine eval-driven flywheel (LLM-judge rubric + one-sided Welch t-test promotion + post-graduation regression watch). The one material gap is on the INPUT side: the high-quality prompt-injection/jailbreak detector (core/screen) is wired only to the agent's own proposal output, not to the untrusted incident/ticket/precedent text that seeds the agent — the exact surface it was built to protect. Secondary soft spots: the eval corpus is thin (flat fixture, reasoning layer under-exercised) and the model.Resolver routing abstraction is dead code (model tiers are hardcoded strings). Multi-agent orchestration is N/A by deliberate single-agent design.

## Top gaps

- Wire the prompt-injection/jailbreak screen onto the untrusted INPUT (alert summary, tracker issue body, retrieved precedent) before it seeds the agent — today core/screen runs only on the agent's own proposal output
- Broaden the eval corpus and add a held-out set so the eval-driven skill-graduation flywheel is exercised against realistic, statistically meaningful data rather than a thin fixture
- Wire or remove the dead model.Resolver so per-component model routing and org-policy overrides are centralized instead of hardcoded "fast"/"primary" strings
