# Anthropic — Effective Context Engineering for AI Agents

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Anthropic** · TG adherence: **2.7 / 5** · Practices audited: **7** (implemented 3 · partial 2 · gap 2 · N/A-by-design 0).

## Verdict

TG's context-engineering is strong on the durable MEMORY plane and prompt LAYERING, but weak on the RETRIEVAL and long-horizon-context practices the source foregrounds. It nails persistent external memory retrieved on demand with an outcome-labelled write path that only learns from confirmed-clean sessions (hygiene that exceeds the source), and it layers the prompt deterministically by execution class with hard size caps on injected skills. But the wired retriever is a single-pass lexical top-3 that filters to score>0 — the OPPOSITE of "maximize recall first, then precision" (provisioned pgvector is unwired); there is no recall-optimized structured compaction anywhere (the loop appends observations verbatim, uncapped, up to the cycle limit, and the model client sets no token budget at all); and agent note-taking is orchestrator-captured provenance, not an agent-maintained scratchpad. Bounded ≤10-cycle triage sessions keep the missing compaction from biting today, but the DeepInvestigation exec-class and Phase-2 will need it.

## Top gaps

- Wire semantic/hybrid recall-first retrieval over the provisioned pgvector (replace single-pass lexical top-3)
- Add recall-optimized structured compaction for DeepInvestigation/Phase-2 long runs
- Enforce a token/context budget in the agent loop and cap tool-output ingestion
- Add an agent-maintained scratchpad/notes for multi-step investigations
