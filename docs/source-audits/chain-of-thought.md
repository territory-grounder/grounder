# Chain-of-Thought

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Paper** · TG adherence: **3.4 / 5** · Practices audited: **10** (implemented 3 · partial 4 · gap 2 · N/A-by-design 1).

## Verdict

On TG-38's headline question, TG nails the INV-08 half exemplarily — the native Go ReAct loop treats every model token as typed, validated, delimited DATA and dispatches only by exact-switch on a strict directive grammar, with deterministic citation and trajectory gates and confidence-as-data fail-closed (agent/loop.go, spec/011); this exceeds the source, which never raises the concern. It genuinely uses ReAct interleaving and strong decision-level auditability (rationale, grounded-stop conclusion, evidence_ids, reconstructable skill provenance, hash-chained ledger). Where it falls short of the source's core CoT thesis: the JSON-only output contract forbids prose and the directive has no thought field, so explicit step-by-step reasoning is compressed rather than emitted; and the advanced reasoning family (Tree-of-Thought, self-consistency, adaptive per-incident thinking budget, multi-model debate) is largely absent, with self-correction deliberately handed to mechanical verifiers rather than the model. PALM is correctly N/A-by-design.

## Top gaps

- Add an optional typed thought/reasoning field to the agent directive (untrusted DATA, never dispatched) so the model emits its explicit step-by-step chain before acting — closes the core CoT gap within INV-08 and improves auditability
- Scale inference budget by execclass (DEEP_INVESTIGATION earns more cycles / a reasoning tier) instead of pinning the fast tier + fixed limits for every incident
- Add self-consistency for high-severity incidents: sample N proposals and require agreement before the gate
- Add a bounded Tree-of-Thought / multi-hypothesis exploration for high-ambiguity incidents, each branch typed under INV-08
