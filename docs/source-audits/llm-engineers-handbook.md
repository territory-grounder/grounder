# Iusztin & Labonne — LLM Engineer's Handbook

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Book** · TG adherence: **3 / 5** · Practices audited: **22** (implemented 6 · partial 8 · gap 5 · N/A-by-design 3).

## Verdict

TG is a strong, often-exemplary realization of the handbook's GOVERNANCE and LLMOps-discipline doctrine — it meets or beats the book on the human "red button" (fail-closed MutationGate + never-auto floor), input/output guardrails (INV-08: no model token becomes control flow, citation gate, grounded-stop, deterministic verifier), a versioned prompt/skill artifact registry with a genuine graduate-by-A/B-trial flywheel (Welch t-test, 3-set offline gate + a sealed holdout it may never tune to, trial-starvation guard), a hash-chained audit-lineage spine, a model-agnostic gateway, and a least-privilege microservice deployment split. It falls short precisely where the book invests most: the ENTIRE RAG feature + inference pipeline is unbuilt in code — retrieval is a transparent lexical scorer, there is no embedding/vector column, no ANN index, no chunking, no cross-encoder rerank, no query rewrite/self-query, and no RAGAS — even though the docs (ARCHITECTURE.md) promise pgvector embeddings + 5-signal RRF (TG's own SYSTEM-MAP.md:92 admits the gap). Full-trace prompt monitoring is also a gap (the Langfuse module is registered but its Record() is never wired into the runner and captures only step-NAME skeletons, and the gateway discards token/cost), and the LLM judge is single-judge/absolute with no pairwise/jury/human-label calibration. Fine-tuning/DPO/QAT are correctly N/A-by-design (closed-weight providers behind LiteLLM; ADR forbids fine-tuning).

## Top gaps

- Build the pgvector RAG plane (embedding write/read path + ANN index) behind the existing Retriever interface — the docs promise it, the code is lexical-only
- Add the advanced-RAG retrieval stages the architecture already commits to (query rewrite/multi-query + cross-encoder rerank + LongContextReorder) once the vector path exists
- Wire full-trace CONTENT prompt monitoring (seed prompt + skill versions + retrieved precedent ids + per-turn directive/answer + token counts) into the runner via the registered Langfuse module, and export live quality/cost/latency series with threshold alerts
- Harden the LLM judge (pairwise + 2-model jury + randomized order + human-label TPR/TNR calibration) and stamp the resolved model id per session
