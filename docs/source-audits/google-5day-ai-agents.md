# Google 5-Day AI Agents Intensive

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Course** · TG adherence: **3.7 / 5** · Practices audited: **8** (implemented 4 · partial 4 · gap 0 · N/A-by-design 0).

## Verdict

TG is a genuinely strong realization of the course's harness-first, spec-driven, graph-grounded, human-as-circuit-breaker thesis — and on several axes it exceeds the predecessor's own gap analysis: the spec lattice actually governs TG's own safety-critical code via a CI-enforced hash lockstep (the predecessor had no spec for itself), and containment is realized structurally (gate-on-structure, argv-only, fail-closed MutationGate) rather than by command blocklists. Where it falls short is exactly where the course sets its highest bar: trajectory-aware evaluation over OpenTelemetry is a documented-but-unwired capability (ExportSpans has no live caller; the LLM judge grades the session record, not the ordered tool path), and the context-as-perimeter "cage" for Phase-2 mutation (egress/exfil control, JIT/ephemeral downscoped credentials, zone-based egress) is an acknowledged gap. Progressive disclosure is approximated (execclass-gated Go-embedded skills, no semantic router / trust tiers) even though the versioned skill-store + graduation flywheel is excellent, and the self-improving loop improves prompts/skills but never feeds back into the spec and runs no microagent squad. Net: a top-tier architecture that meets or beats the course on 4 of 8 practices and is honestly partial on the other 4, mostly on the not-yet-lived Phase-2 runtime.

## Top gaps

- Wire trajectory-aware evaluation over OpenTelemetry (export session spans + judge the ordered tool path + land the deterministic trajectory veto that overrides the LLM judge)
- Build the Phase-2 context-as-perimeter cage: egress/exfil allowlist + JIT/ephemeral downscoped credentials (zero ambient authority) + zone-based egress for the mutating actuator
- Add a semantic/hierarchical skill router plus skill trust-tiers, a skill-inspector scan, and a library-level skill-collision eval to the skill store
- Close the self-improvement loop back into the spec (a super-architect that feeds mined failures into requirements, not just prompt/skill bodies)
