# ReAct (Reason + Act)

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Paper** · TG adherence: **4 / 5** · Practices audited: **10** (implemented 6 · partial 4 · gap 0 · N/A-by-design 0).

## Verdict

TG's agent/loop.go is a faithful, rigorously-hardened implementation of the core ReAct cycle: it interleaves allowlisted tool actions with orchestrator-captured observations that are fed back into the model's next turn, the loop is bounded (poll/hard-halt limits), and it corrects errors mid-run via re-prompt gates and a deterministic trajectory veto. On grounding it EXCEEDS the base paper — observations are trusted DATA (never model free-text) and a mechanical citation gate (REQ-1007) forces post-observation proposals to cite real captured evidence. Where it falls short of full ReAct fidelity: the strict-JSON directive protocol deliberately suppresses the explicit interleaved reasoning trace (no `thought` field is parsed or persisted; only the terminal rationale survives), a malformed directive fails the whole session closed instead of self-correcting, there is no confidence-vs-reasoning self-consistency check (the predecessor had one), and the computed trajectory is used only for the loop-veto and never persisted for post-hoc scoring. Net: the Reason+Act mechanics are strong; the observable/auditable reasoning-trace and self-consistency layers are the thin spots.

## Top gaps

- Capture and persist a per-step reasoning trace (optional untrusted `thought` DATA field) for audit and eval, without it becoming control flow
- Add a bounded format re-prompt so a malformed directive self-corrects instead of terminating the whole session
- Add a confidence-vs-reasoning self-consistency check (high-confidence + hedging / uncited rationale -> escalate)
- Return and persist the ordered trajectory step-sequence to the ledger for post-hoc trajectory scoring
