# Reflexion

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Paper** · TG adherence: **2.3 / 5** · Practices audited: **8** (implemented 1 · partial 4 · gap 2 · N/A-by-design 1).

## Verdict

TG has genuine strengths adjacent to Reflexion: cross-episode persisted memory (core/lessons distils resolved incidents into knowledge.Incident, the knowledge retriever surfaces them, and temporal/runner/activities.go:263-269 re-injects them into the agent seed as PRIOR PRECEDENT), a real and separate Generator-Critic (the core/judge LLM judge scoring five dimensions, read-only over the record), and a system-level improvement flywheel with statistical A/B trials plus a post-graduation regression watch. But it is fundamentally NON-reflective in the Reflexion sense: it never produces a verbal self-reflection over a failed trajectory, it learns ONLY from confirmed-clean successes (core/lessons/lessons.go:39 deliberately excludes deviations/partials/unconfirmed to avoid corpus poisoning), and the critic's verbal feedback dead-ends because the flywheel generator (core/skillstore/generate.go:59-66) is driven by a numeric regression signal alone, never the judge comment or the failing session. The defining Reflexion move — reflect on a failure, persist the lesson as verbal reinforcement, reuse it on the next attempt — is absent; TG substitutes blind statistical skill improvement.

## Top gaps

- Generate and persist a verbal self-reflection on failed/escalated/deviated trajectories (the missing Reflexion memory)
- Learn from failures/near-misses into a caution lane, not only confirmed-clean successes
- Close the critic loop: feed judge comments + failed trajectories into the skill-flywheel generator instead of a numeric-only regression signal
- Add an offline reflect-and-retry eval mode to measure whether reflection improves triage outcomes
