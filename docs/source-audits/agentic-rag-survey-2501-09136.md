# Agentic RAG Survey (arXiv 2501.09136) — Self-RAG / CRAG / GraphRAG / HyDE

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Paper** · TG adherence: **2.3 / 5** · Practices audited: **6** (implemented 0 · partial 3 · gap 3 · N/A-by-design 0).

## Verdict

TG's retrieval plane is a single-shot, transparent LEXICAL retriever (core/knowledge/retriever.go) that injects the top-3 prior incidents into the agent seed once (temporal/runner/activities.go:451-458); it has no embeddings, no hybrid RRF, no retrieval evaluator, and no re-retrieval, so three of the survey's named practices (CRAG-style corrective retrieval, HyDE, retrieval self-correction) are outright gaps. Where TG genuinely meets the survey is adjacent-but-real: it has a first-class, wired ENTITY-RELATION graph — the estate causal graph (core/estate + get-estate-context tool) — that satisfies GraphRAG for the topology domain, and an agent-decided ReAct evidence-gathering loop with a mechanical citation/grounding gate (REQ-1007) + grounded-stop (REQ-1008) + confidence gate that realizes the SPIRIT of Self-RAG's supported/useful reflection in an INV-08-safe way (stronger than reflection tokens for grounding). But the knowledge CORPUS itself is thin: it is not graph-linked, not self-corrected, and not agent-triggerable, and the highest-value GraphRAG query ("what past incidents hit the hosts in this alert's blast radius") is unbuildable because the precedent corpus and the estate graph are disjoint planes.

## Top gaps

- Graph-link the incident-precedent corpus into the estate graph (build the incident-knowledge GraphRAG) so blast-radius precedent is a graph walk
- Add a retrieval-quality evaluator + agent-triggerable re-retrieval loop (CRAG / retrieval self-correction) that consumes Hit.Score and reformulates the query
- Add Self-RAG-style relevance reflection over retrieved precedent (agent marks used-vs-rejected hits as DATA) to down-rank stale precedent without violating INV-08
- Wire the ADR-0003 pgvector embedding/hybrid-RRF plane, which unlocks HyDE for the free-text summary channel
