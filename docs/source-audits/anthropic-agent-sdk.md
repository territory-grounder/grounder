## Verdict

TG's native Go ReAct loop faithfully realizes the source's gatherŌåÆactŌåÆverifyŌåÆiterate spine and, on verification, materially exceeds it: a deterministic sole-author verdict (INV-10), a falsifiable machine prediction committed before the action, an inline citation/grounding gate, a trajectory veto, and a shared 5-dimension LLM-judge rubric ŌĆö verification is the article's "key to reliable agents," and TG treats it as first-class. Where it falls short of the source is on the two other named focal patterns: sub-agent delegation is claimed in the constitution (DEEP_INVESTIGATION) but not built ŌĆö loop.go is single-agent and repurposes the handoff limits as a turn budget ŌĆö and MCP is scaffolded + security-designed (ADR-0012, interceptor + pinned hashes) but disabled in Phase 0/1, so no MCP tool is exercised in the loop. Context-gathering is real via read-only native tools, but the retrieval plane is lexical + empty-by-default (pgvector not wired to the agent) and there is no in-loop compaction. Several SDK affordances (model-generated code/bash as an action surface; autonomous verifyŌåÆre-plan) are correctly rejected by design, not gaps.

- Build binding depth-1 read-only sub-agent delegation (or downgrade the DEEP_INVESTIGATION constitution claim) to close the sub-agent spec-vs-code gap

- Wire the pgvector semantic retriever behind the Retriever interface and populate the precedent corpus (today lexical + empty by default)
- Add recall-optimized context compaction in the loop before the window fills (currently append-only, bounded only by the 10-cycle hard-halt)
