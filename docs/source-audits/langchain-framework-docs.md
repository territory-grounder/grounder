# LangChain — Framework Docs

> **[O] audit-overlay · TG-38 source benchmark.** Category: **LangChain** · TG adherence: **3.6 / 5** · Practices audited: **18** (implemented 9 · partial 5 · gap 2 · N/A-by-design 2).

## Verdict

TG is strong exactly where LangChain says it matters most: context engineering is a first-class, deterministic discipline (per-execution-class skill composition + RAG precedent/CMDB/ticket seed assembly), and its safety-flavoured HITL (fail-closed MutationGate + POLL_PAUSE band + never-auto floor + sealed-action approval vote) exceeds the middleware's approve/reject checkpoint. Skills, procedural/semantic memory, model-fallback (delegated to LiteLLM), and stopping conditions (cycle hard-halt + trajectory veto) are all present and robust. It falls short on the runtime "context management" middleware family: there is NO context-editing/summarization (message history is append-only within the cycle cap), NO tool-retry with backoff (any tool error fails closed), PII redaction only covers outbound human notices (not model-bound tool outputs or the ledger), and the tool set is fixed (no dynamic per-task tool filtering). Multi-agent orchestration (subagents/agent-to-agent handoffs) is deliberately absent by the single native Go loop / least-autonomous topology, so those are N/A rather than gaps; router and RAG-as-agent are only partially realized.

## Top gaps

- Add context-editing/summarization to the ReAct loop — trim/summarize older OBSERVATION payloads so long investigations stay within token budget (currently append-only)
- Add tool-retry with exponential backoff + jitter for transient read-only tool failures (currently any tool error fails closed to stop)
- Extend PII/credential redaction to model-bound tool outputs and the audit ledger, plus IP/MAC rules (today only outbound human notices are scrubbed)
- Add dynamic per-task tool filtering keyed on execution class (LLMToolSelector analog) — the tool set is currently fixed regardless of FAST vs DEEP
