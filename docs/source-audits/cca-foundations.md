# Claude Certified Architect (Foundations)

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Cert** · TG adherence: **3.8 / 5** · Practices audited: **10** (implemented 6 · partial 3 · gap 0 · N/A-by-design 1).

## Verdict

TG faithfully implements — and by making them STRUCTURAL/binding rather than conventional, arguably exceeds — the cert's safety-critical sub-agent principles: the 3-tier hierarchy (deterministic Temporal orchestrator + fast tier-1 suppression → untrusted model loop → human), least-privilege read-only tool allowlists with write tools structurally withheld, single-grammar structured output, obstacle-reporting via confidence/escalate/grounded-stop discipline, no-ungrounded-authority via the citation gate (INV-08/REQ-1007), and HITL-for-every-infra-change (mutation OFF, POLL_PAUSE zero-value, deny-by-default vote wait, non-configurable never-auto floor). Where it falls short of the cert's letter: multi-agent parallel delegation is deliberately de-scoped (defensible, live-vindicated) but the CONSTITUTION/ARCHITECTURE still CLAIM a manager-with-sub-agents DEEP_INVESTIGATION pattern that agent/loop.go does not build — a self-admitted spec-vs-code gap. Complexity routing is decided-but-not-acted: execclass.Classify picks 5 classes yet its differentiating helpers (SkipsAgent/NeedsDeepContext) are unwired, execClassFor feeds only one signal, and the agent model is hardcoded "fast" regardless of class. Context isolation holds at the session and control-plane layers but has no sub-agent-level isolation, compaction, or handoff envelope.
