# Effective Harnesses for Long-Running Agents (Anthropic)

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Anthropic** · TG adherence: **3.8 / 5** · Practices audited: **9** (implemented 4 · partial 4 · gap 0 · N/A-by-design 1).

## Verdict

TG's durable-execution core is genuinely strong and beats most home-grown harnesses: the session Runner IS a Temporal workflow, so state survives worker crashes/deploys, a POLL_PAUSE session pauses on a durable 24h timer + signal channel and resumes deterministically, in-flight histories are protected across code changes with GetVersion change markers, and every step budget is bounded (10-cycle loop, trajectory veto, 15-min activity cap, 64-vote flood cap). Where it falls short of the source's "long-running AGENT" advice is granularity and multi-session continuity: the entire ReAct loop runs inside ONE non-durable activity with no per-turn checkpoint (a mid-investigation crash re-runs the whole loop, not resume-from-turn-N), there is no activity heartbeat on that long activity, and continue-as-new is documented as implemented (EXTERNAL-AUDIT-LESSONS.md:69, TESTING-AND-BENCHMARK.md:175) but is never actually called anywhere in the tree. Cross-session coherence exists only at the fleet/knowledge level (retrieval of prior incidents + durable skill-store seeds + judge records), not as a single task carried across sessions — which is largely N/A for bounded triage, but the doc claims of "per-turn snapshots" overstate what the code does.

## Top gaps

- Per-turn durable checkpoint of the agent ReAct loop (single non-durable activity re-runs the whole investigation on crash; make the 'per-turn snapshots' doc claim real)
- Reconcile the continue-as-new doc overclaim (EXTERNAL-AUDIT-LESSONS.md:69, TESTING-AND-BENCHMARK.md:175) with reality — correct the docs or implement it where genuinely needed
- Add activity heartbeat + HeartbeatTimeout to the long InvestigateActivity so a hung investigation is detected before the 15-minute hard cap
- Wire the learned cross-session co-occurrence feed to the live ingest stream to close the fleet-level multi-session coherence loop
