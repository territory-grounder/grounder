# Gulli ‚Äî Agentic Design Patterns (21 patterns)

> **[O] audit-overlay ¬∑ TG-38 source benchmark.** Category: **Book** ¬∑ TG adherence: **3.9 / 5** ¬∑ Practices audited: **21** (implemented 12 ¬∑ partial 7 ¬∑ gap 0 ¬∑ N/A-by-design 2).

## Verdict

TG is genuinely strong on the safety-and-governance half of Gulli's catalogue and in several places exceeds the predecessor: guardrails (fail-closed MutationGate + immutable never-auto floor + jailbreak screen + INV-08 "no model token becomes control flow"), planning (a deterministic Temporal investigate‚Üíclassify‚Üígate‚Üíverify workflow with a content-hashed ActionManifest), goal-setting/monitoring (a committed falsifiable prediction diffed by a deterministic verifier, plus a judge-liveness + frontier drift/death cross-check the predecessor only added late), human-in-the-loop (an action_id-bound vote loop with deny-by-default and DoS-bounded polls), evaluation (shared judge rubric + Welch-gated A/B skill trials), tool-use and learning (a real generate‚Üíadmit‚Üítrial‚Üígraduate flywheel). It falls short on the "intelligence-amplifier" half: RAG is lexical-only despite pgvector being in the stack (the exact Ch.14 / predecessor GAP-1 weakness), reasoning is ReAct-only with no tree-of-thought/self-consistency/step-back, MCP is a scaffolded loadable actuation surface rather than a live consumed client fleet, there is no per-session token/cost budget ceiling, and no intra-task parallelization or proactive estate exploration. Multi-agent and A2A are N/A by deliberate single native-agent, single-org topology.

## Top gaps

- Semantic RAG: wire a pgvector/embedding retriever behind the Retriever interface (currently lexical-only; pgvector unused) ‚Äî the Ch.14 / predecessor GAP-1 quality gap
- Reasoning techniques: add tree-of-thought multi-hypothesis + self-consistency + step-back for deep/correlated incidents (ReAct-only today)
- Resource budget enforcement: per-session token/cost ceiling + daily budget with plan-only degradation (no in-app budget enforcement)
- Live read-only MCP client: consume external MCP servers as investigation tools behind the interceptor (the MCP surface is a registry scaffold, not a consumed client)
