# Equipping Agents with Agent Skills

> **[O] audit-overlay · TG-38 source benchmark.** Category: **Anthropic** · TG adherence: **2.5 / 5** · Practices audited: **5** (implemented 1 · partial 1 · gap 1 · N/A-by-design 2).

## Verdict

TG's skill store (spec/014) is a genuinely strong, versioned, provenance-tracked skill system that meets or exceeds the source on skill COMPOSITION — multiple applicable skills are composed into one seed deterministically, in order, with override-by-name, a pinned floor, a total fallback, and a per-session skill_load record that makes the seed byte-reconstructable. But it deliberately diverges from the source's two load-bearing ideas: progressive disclosure is only a coarse analog (a pure typed-signal selector gates whole skill bodies by phase x exec-class — it achieves the "keep irrelevant instruction out of the prompt" GOAL, but there is no model-visible name/description index, no on-demand L2 pull, and the full 8 KiB body is injected whenever a skill matches). The SKILL.md format is asserted (ADR-0012 claims rows map 1:1 to frontmatter+body) but NOT implemented — no serializer, no frontmatter parser, no export, and no `description` frontmatter key exists anywhere in the schema or HTTP surface. Code bundling (L3) and model-driven skill discovery are correctly N/A-by-design: bundled executables are stripped as a supply-chain/INV-08 hazard and all actuation flows only through the argv-only governed interceptor.

## Top gaps

- Implement SKILL.md import/export (YAML frontmatter <-> skill_version row) + add the canonical `description` field, making the ADR-0012 '1:1 mapping' real rather than aspirational
- Progressive disclosure within a skill: always-load a short summary/trigger line and defer the full body until the matching alert-class/phase, instead of injecting whole 8 KiB bodies on exec-class match
- Build the console skill-library read surface (spec/014 REQ-1313 / frontend/src/panels/skills) — no frontend panel exists today; the versioned library, rationale log, and composed-seed provenance are backend-only
