Citations are into the predecessor (`scripts/…`, `workflows/*.json`) or TG (`core/…`, `temporal/…`, `agent/…`); live findings are the audits' verified DB queries.*

## 1. WHAT THE COMPLETE SYSTEM IS

The predecessor is a **closed-loop autonomous SRE platform**: an alert becomes an investigated, optionally-remediated, verified, and learned-from incident, with a human wired in as a circuit-breaker rather than an operator-in-the-loop. Eleven subsystems compose into **four planes plus a cross-cutting utility layer**, and the single most important architectural fact is that the whole thing runs on a **two-lane fail model — the advisory/triage lane fails OPEN, the remediation lane fails CLOSED** (`PORT-FIDELITY-AUDIT.md:183`).

### The one-incident data/control flow

```
alert/webhook/CrowdSec/scanner
   │
   ▼  [ADMISSION plane — fail-open, runs in code BEFORE any model spend]
 receiver: normalize → dedup → flap → burst → correlate           (Area 5 ingest)
   │
 Tier-1 suppression: dedup → blast-radius FOLD → known-pattern → active-memory
   │                         └── consults the estate causal graph (§infragraph)
   │  critical/unknown ALWAYS escalate
   ▼ escalate → YouTrack issue (correlation key = external_ref/issue_id)
 YouTrack Receiver webhook  ── async fire-and-forget ──▶ Runner
   │
   ▼  [DECISION + REMEDIATION plane — fail-closed]
 Runner (n8n, 53 nodes):
   slot lock (600s) + 30s cooldown
   → RAG context  ─────────────▶ 5-signal RRF over incident_knowledge / transcripts /
   │                              wiki / chaos / diary  (KNOWLEDGE plane)
   → commit PREDICTION ────────▶ estate blast-radius, plan_hash-keyed, computed OUTSIDE the LLM
   → classify RISK (3-band) ───▶ + inline jailbreak screen → HIGH ⇒ POLL_PAUSE (SECURITY)
   → build prompt ─────────────▶ team charter (advisory) + model/opus hint (MODEL/COST)
   → launch bg `claude` process (systemd-run, timeout 300–600s, 12G cap)
        ├── Progress Poller (30s) tails JSONL, runs session-tripwire  ⇄  Matrix Bridge
        │                                                                  human-in-loop:
        │                                                        !session pause/resume/done/
        │                                                        cancel + SIGINT mid-flight inject
        └── territory GATE fires PreToolUse on every WRITE ─────▶ zone rulebook must be loaded
   → mechanical VERDICT (match/partial/deviation) ── acting LLM never scores itself
   → band-aware reconciler → YouTrack transition "To Verify"  (the SINK)
   │
   ▼  [ASSURANCE plane]
 Session End (18 nodes): LLM summary+LESSON → trajectory score → LLM-JUDGE ⇄ frontier crosscheck
   → populate incident_knowledge + lessons_learned + GraphRAG → OTel export
        chaos drills kill tunnels, commit a cascade prediction, verify forward, and
        fold verdicts into estate-graph routes_via dynamics (the graph "earns" its numbers)

 CROSS-CUTTING: every LLM call → model routing → circuit breaker → cost ledger.
                every decision → typed event_log row + governance hash-chain + per-turn snapshot.
```

### The eleven subsystems, grounded in live findings

**1. Orchestration / session lifecycle (n8n).** Five cooperating workflows over host-local sentinel files, `/tmp` JSONL, and a SQLite state DB run the whole incident session: YouTrack Receiver → Runner → Progress Poller → Matrix Bridge (human-in-loop) → Session End. Live: **1,109 sessions, 17,491 turns, $1,622.62** spend; num_turns clusters at 11–40 with a tail to 143. This is the spine; three of its five workflows are the operator-facing value (steering + progress + sink). Note the **counting-semantics trap**: "turn" means three different things — Stop-hook turns (`session_turns`, live max=**4**), assistant-message turns (`session_log.num_turns`, live max=**143**), and typed `event_log` events.

**2. GraphRAG knowledge graph.** A `graph_entities`/`graph_relationships` retrieval graph (incident/host/alert_rule/lesson/chaos nodes; edges affects/triggers/resolves/chaos-tests) queried by an LLM-planned recursive-CTE `traverse`. **Critical finding: this is NOT the production RAG.** Every workflow and OpenClaw triage skill calls only `kb-semantic-search.py search` (the 5–6-signal RRF vector pipeline); `graph`/`traverse` appear only in tests/evals. `bloom.py` is Bloom's *educational* taxonomy, not a bloom filter. The graph-walk is real, first-class, and reaches production context in **zero** workflows.

**3. Infragraph / estate causal graph.** The crown jewel: ONE causal graph in the *same two tables* as GraphRAG, with a 1:1 `infragraph_dynamics` sidecar, edge convention `SOURCE depends-on TARGET`, `blast_radius(H)` walking edges INTO H. Multi-source truth is confidence-graded (tunnels 1.00 > live PVE 0.95 > LibreNMS/NetBox 0.90/0.85 > declared 0.85 > learned ≤0.75, capped below the 0.80 suppression cutoff), edges self-expire at 7d, entity resolution is mandatory before every edge write, and blast radius uses path-product confidence + common-cause siblings (0.6×) + a Laplace(1,4)/τ=0.10 cascade gate + a real degree-preserving shuffled negative control with a `control_ratio ≤ 0.5` falsifiability gate (`PORT-FIDELITY-AUDIT.md §1`). **Raw structural precision is ~0.05–0.15 by design** (live: action 0.056 vs shuffle 0.007). This is the substrate the fail-closed prediction gate reasons over.

**4. Agent-teams / handoff / A2A.** Four surfaces meant to turn an implicit sub-agent graph into a "governed" multi-agent system: `team_formation.py` (rule-based charter), `agent_as_tool.py` (spawn-with-depth-ceiling), `handoff_depth.py` (depth/cycle rails: POLL@5, HALT@10), the `HandoffInputData` envelope, and the NL-A2A JSON-RPC protocol. **Live reality: almost entirely inert.** Max handoff_depth ever reached = **2** (thresholds 5/10 never fired), `handoff_log` = 0 rows, `team_charter` = 0 events, `agent_as_tool_call` = 0, `a2a_task_log` = 53 completion-only rows abandoned after two weeks. `from_agent='unknown'` on all 83 handoff events. The read-only researcher sub-agents have no Task tool, so depth is structurally capped at ~1–2 — the rails are **unreachable by construction**.

**5. Eval / benchmark / judge.** The forward-only, no-human quality layer: LLM-judge (5 dims, local gemma3:12b ~$0) → jury blend → **composed verdict where a DETERMINISTIC 8-boolean trajectory scorer HARD-VETOES the LLM judge** → frontier cross-check (model-independent anchor) → GEPA + prompt-patch A/B trials → RAGAS → 3-set flywheel (regression/discovery/sealed-holdout). **The dead-judge saga is live-visible**: the local judge was ~dead for 7 weeks (unscored fraction 76–77% in May–June) yet kept writing `-1` rows, so nothing looked dark; only `judge_crosscheck` (8/17 rows caught DEAD-LOCAL) would have found it. `prompt_patch_trial` promoted **zero** winners in 9 trials.

**6. Teacher / learning / lessons.** TWO decoupled loops: (a) the **machine loop** — resolved incidents → `lessons_learned` (27 curated rows, `resolves@1.0` + `affects@0.7`) + `incident_knowledge` (2,633 embedded rows, the real RAG feedback corpus); (b) the **human loop** — a Matrix SM-2/Bloom teacher-agent tutoring operators on agentic-systems theory, behind a hallucination gate (every snippet must be a verbatim substring of source) and a confidence-clarifier gate. The teacher **never writes back** to lessons/graph — it is an operator-education side-channel, barely exercised (`learning_progress` = 1 row).

**7. Chaos engineering.** Real production failure drills (kill tunnels/NICs/containers) under a 6-layer safety envelope, each a labeled `chaos_experiments` row (162 rows, 146 PASS), folded back into the graph. **The advertised "chaos verdicts strengthen edge confidence" is inert** — all 18 `source='chaos'` dynamics rows sit at confidence **1.0** because chaos only touches tunnel `routes_via` edges which are declared at 1.0 and MAX-ratcheted. What chaos *actually* contributes is **provenance + empirical dynamics** (observation_count up to 31, delay p50/p95, recovery p50) — the "earned" measured layer. The most portable piece is the **3-layer alert FREEZE** (Alertmanager silence + LibreNMS maintenance + a state file the receivers check) so a drill's expected alerts don't spawn remediation.

**8. Memory / session-state.** Three layers: `snapshot.py` (pre-mutation RunState), `session_events.py` (18-type typed `event_log`, WAL+8-retry emit), and the long-term memory store (`incident_knowledge`, `session_transcripts` = 27,865 rows / 4th RRF signal, `agent_diary`, `openclaw_memory` = 6,731 rows). **Live: the crash-resume mechanism is capture-only** — 3,037 snapshots written, `rollback_to()` never called in production, `turn_id=-1` on 100% of rows, 54% carry no restorable state. Of 18 event types only 9 ever fire; `tool_started`=0 (only `_ended` fires); the `mcp_approval_response` vote ledger = 0; `tier1_suppression` (1,012) bypasses the typed emitter via a second direct-SQL writer.

**9. Territory (the namesake).** A "territory" is a governed **operating-DOMAIN** — 8 static infra zones (k8s/network/edge/pve/native/docker high-stakes; ci/images not), each with a load-bearing `CLAUDE.md` manual. `territory.py` resolves {hostname, command, path, VMID} → {zone, role, is_stateful, high_stakes, the manual chain} via a confidence ladder (path 0.95 > command 0.85 > entity-type 0.85 > host-heuristic 0.80), fail-safe (unplaceable → territory=None → no-op). `territory-gate.py` HARD-BLOCKS a write in a high-stakes zone until that zone's manual has been Read+acked *this session*. Two load-bearing mechanics: **stateful-target dominance** (`qm reboot <k8s-ctrlr>` is governed by k8s quorum rules, not pve verb-rules) and a separate sibling **egress guard** (exfil allowlist). The name is a double meaning — Korzybski's "the map is not the territory": GROUND the model's map against reality.

**10. Security / secops.** Four defensive layers: a pure-regex **jailbreak detector** (5 derailment classes, Unicode-hardened, deliberately Greek-first-class) wired inline to force HIGH→POLL_PAUSE fail-closed; a **SQLite-persisted 3-state circuit breaker** (the defining property is *cross-process* shared state so Runner/Poller/cron see one view of upstream health) guarding every RAG call; a dark-first **intermediate topic rail**; and **ChatSecOps/CrowdSec** IPS feeding a triage→escalation→YouTrack path with a 6-hourly auto-suppress learn-loop. Live: the jailbreak detector's *only* production signal was a **false-positive wave** (55× lost-in-middle-bait, then fixed by SCOPING the input to untrusted narrative, not by changing regexes); the intermediate rail fired **once, ever**.

**11. Model orchestration / cost.** A 3-plane fleet: a subscription Claude-Code plane (flipped by rewriting `settings.json`, live default Z.ai/glm), a paid per-token plane (Mistral+DeepSeek via a *reused* external LiteLLM), and a local Ollama plane ($0). `config/model-routing.json` is the nominal SSOT; cost lands in one `llm_usage` table. **Live: the "one rate card" is a fiction** — the highest-spend writer (the n8n runner) hardcodes Anthropic-Opus rates inline; ≥3 divergent cost paths coexist; **a $0 local model (gemma3:12b) is booked at $1,414, of which $771.91 is priced at Opus list rate**; per-component attribution was never shipped (no `component` column exists despite the config claiming it).

### How they interconnect into one platform

- **The estate graph is the spine that the prediction gate, Tier-1 blast-fold, and chaos all read/write.** Suppression folds against it; the Runner commits a prediction over it; chaos drills validate its edges forward-only; the verdict scores against real alerts.
- **The knowledge plane (RAG) feeds every session's context**, and Session End closes the loop back into it (`incident_knowledge`/`lessons_learned`/GraphRAG) — "a session is done only when its knowledge is written back" (`CAPABILITY-INVENTORY.md:68`).
- **The seam that defines the port:** GraphRAG (incident/lesson/chaos/`host`) and the estate infragraph (`lxc`/`vm`/`pve_node`/`tunnel`) **share the same two tables and even the same machine NAMES but are topologically disjoint** — 27/35 GraphRAG `host` names also exist as typed infragraph nodes, yet the verified count of bridging edges = **0**. The highest-value question ("what past incidents hit the hosts in this alert's blast radius?") cannot be answered by a graph walk; it works only because the vector RRF path sidesteps the graph entirely.
- **Cost/model, circuit breakers, `event_log`, the governance hash-chain, and per-turn snapshots are the cross-cutting utilities** every plane calls. They are where the fragmentation lives (split telemetry, ≥3 cost paths, two writers into one event table).

## 3. NET-NEW CAPABILITIES TG LACKS (ranked)

Ranked by value to a *governed* rewrite × risk of shipping without. (The estate-graph builder itself — P0-1 — is the prerequisite for #4/#5 and is covered in the sister audit; it is assumed as the foundation these build on.)

**1. Persisted, cross-process circuit breaker.** *Buys:* bounded-failure short-circuit on every model-gateway/judge/RAG call, with half-open recovery and state shared across pods/goroutines through Postgres exactly as the predecessor shared it through SQLite (`circuit_breaker.py:12-14`). *Risk without it:* this is the exact outage class behind the 3-week dead-judge incident. TG's LiteLLM fallback *ladder* retries every rung on every call — no persisted OPEN state, no bounded failure — so a sustained provider outage keeps getting hammered by every worker (`PORT-FIDELITY-AUDIT.md:159/#24`). CONSTITUTION.md:130 promises "named, observable circuit breakers"; there is **zero Go**. Cheapest high-value gap.

**2. Frontier cross-check + deterministic trajectory veto (the eval anchor).** *Buys:* a model-INDEPENDENT liveness anchor that samples the judge-independent `sessions` table (so a judge writing all `-1` rows is still caught), plus a cheap deterministic 8-boolean trajectory scorer that HARD-VETOES the LLM judge and surfaces the judge-fooled/quality-gap disagreement as the primary release signal. *Risk without it:* a single self-referential metric is structurally blind to a dead judge (live: 76–77% unscored for 7 weeks, undetected), and a fooled LLM judge becomes the last word. TG ships judge-death detection but the anchor that actually caught the outage is a README bullet (`eval/README.md:4`). Land both the moment the LLM judge lands.

**3. Territory semantics (server-side zone resolver + policy + stateful-target dominance + egress).** *Buys:* the product's namesake capability — a typed classifier that tags each graph-resolved action with zone/role/stateful/high_stakes and clamps stateful ops server-side, plus the **stateful-target dominance** rule (a pve reboot of a k8s controller is governed by quorum rules, not verb-rules) that a flat op-class classification cannot express, plus an egress exfil allowlist. *Risk without it:* TG's interceptor has only structural checks — no zone concept anywhere in Go. The 11 live k8s-ctrlr/openbao + pbs stateful hosts get no quorum/data-loss clamp; there is no exfil guard. Port the INTENT (zone rules govern, machine-enforced) not the mechanism (did-the-agent-Read-a-markdown-file).

**4. GraphRAG / knowledge-retrieval plane (unified host identity).** *Buys:* the entire triage-context assembly — 5-signal RRF over incident_knowledge/transcripts/wiki/chaos/diary with cross-encoder rerank, confidence-gated synth, and a citation oracle — plus, if wired, a graph traversal that finally answers "past incidents on the hosts in this blast radius" because TG uses **ONE canonical host node** instead of the predecessor's disjoint `host` vs `physical_host` twins (0 bridging edges despite 27/35 name overlap). *Risk without it:* the retrieval/knowledge plane is entirely unbuilt (no vector column, no embeddings, no RRF, no rerank) — the larger unaudited gap. Also port the production-grade robustness scaffolding (per-call breakers, `SEARCH_BUDGET_S` HyDE skip, citation oracle, staleness warnings) that separate a demo RAG from an alert-path-safe one.

**5. Chaos freeze gate + empirical-dynamics trust axis.** *Buys:* (a) a maintenance-window/drill FREEZE modeled as an org-scoped policy row consulted BEFORE per-host resolution (P0-6) — the highest-value graph-independent lift; (b) an "empirically validated N times" trust axis (observation_count + delay/recovery percentiles) distinct from static confidence, feeding the dynamic prediction window. *Risk without it:* TG will spawn remediation for the very alerts a declared maintenance window is expected to cause. Do NOT port the literal verdict→confidence map (live-proven inert on the declared-1.0 tunnel edges chaos touches). Sequence after the estate graph exists.

**6. Grounding gates (hallucination gate + confidence-clarifier).** *Buys:* two domain-agnostic primitives — require every LLM-asserted factual snippet to be a verbatim substring of an approved source (retry N×, then fail to None, never ungrounded), and force a clarifying/escalation step when self-reported confidence is below threshold. *Risk without it:* every LLM-generated artifact (plans, root-cause hypotheses, proposed remediations) can drift ungrounded with no calibrated abstention. Cheap, already slated to be lifted into TG's shared judge/quality library (`CAPABILITY-INVENTORY.md:101`). The *pedagogy* around them is correctly out-of-scope.

**7. Agent-teams / DEEP_INVESTIGATION sub-agent delegation.** *Buys:* a manager-with-read-only-sub-agents-as-tools pattern for deep investigations, with binding depth/cycle rails and propagated parent→child lineage. *Risk without it:* low — the live data vindicates de-scoping (max depth ever 2, thresholds 5/10 never fired, `handoff_log`=0, `team_charter`=0, a2a abandoned). The real risk is the **inverse**: TG's constitution CLAIMS this pattern while `agent/loop.go` implements a single-agent loop and reuses the "Handoff" names for a per-agent turn budget. Either build it binding (loop.go's `ReadOnly()` gate makes it safe) or **downgrade the doc claims to match the single-loop reality** — do not ship the spec-vs-code gap. Build the envelope only when a real second agent needs inherited context.

## 4. TOP CROSS-SYSTEM INSIGHTS

Design principles that recur across the subsystems — these are the transferable IP and the traps.

**I. "Wired-but-disconnected dead capability" is the dominant failure mode — it appears in EVERY subsystem.** GraphRAG traverse (called by zero workflows), `handoff_log`/`team_charter`/`agent_as_tool_call` (0 rows), a2a (abandoned after 2 weeks), `session_state_snapshot.rollback_to()` (never called), the intermediate rail (1 event ever), the jailbreak detector (nightly-only until late), GEPA + prompt-patch trials (0 winners), bi-temporal decay (reporting-only dark), FAISS (never read), per-component cost tracking (aspirational). **The builder's #1 rule: verify what actually ran in production before transcribing impressive-looking machinery; wire end-to-end or delete.** TG's INV-21 (fail-boot-if-unwired self-test) is the structural antidote — and TG is *re-importing the anti-pattern right now* with a prediction gate wired to an empty graph (`PORT-FIDELITY-AUDIT.md:109,177`).

**II. Fail-safe direction is everything, and it is asymmetric by lane.** The advisory/triage lane fails OPEN (an ambiguous alert still investigates); the remediation lane fails CLOSED (unknown ⇒ never-auto). This recurs identically in: prediction eligibility (target-not-in-graph ⇒ `eligible=False`), cascade gating (`drop=True` in the fold lane, **`drop=False` in the action lane** so a dropped host can't flip a real cascade match→deviation), suppression (fails open, critical always escalates), territory (blocks only a *confidently-resolved* high-stakes write), jailbreak (any hit ⇒ POLL_PAUSE), and the eval maxim **"no judgment data is a PROBLEM, never a pass."** Get any one backwards and the safety contract inverts — TG has already inverted it in three places (cross-site exclusion, nil-hook eligibility, model-declared reversibility bool; `PORT-FIDELITY-AUDIT.md:193`).

**III. A self-referential metric cannot detect its own death — you need a model-INDEPENDENT anchor sampling the tables the thing does not write.** The judge died for 3 weeks writing `-1` rows; only the frontier cross-check (sampling `sessions`, not `session_judgment`) caught it. The same shape recurs as dead-man switches, `absent()` "no-data ≠ no-alert" clauses, the canary's live-DB-leak counter, and the counting-semantics traps (three definitions of "turn"; `mcp_approval` events landing in a *different* DB via a defaulted path). Corollary: one authoritative store, never a defaulted path.

**IV. Controls must be BINDING and STRUCTURAL, not advisory and conventional — gate on STRUCTURE, not strings.** The predecessor's controls were "real in intent, false in binding": the handoff-depth hook exits 0 (its 10 cycle detections prevented exactly zero delegations; IFRGRSKG01PRD-269 looped 5×), the territory gate reads a markdown ack, known-transient suppression fires on bare string equality, cost is a convention re-implemented per writer, jailbreak string-parse bugs failed OPEN. TG's antidote is the same everywhere: typed enums whose **zero-value is the fail-closed state** (`Band`=POLL_PAUSE, `Reversibility`=Irreversible), a single unexported `Execute` chokepoint, one `ParseProposal`, required-field audit writes, and cost computed once at a chokepoint. Never a command-string blocklist (wrong layer).

**V. Confidence must be LAYERED, PROVENANCED, and first-class on the row.** Human-curated 1.0 > live 0.95 > monitoring 0.90 > CMDB 0.85 > text-mined ≤0.75 (deliberately capped BELOW the 0.80 suppression cutoff so a heuristic edge can never outrank ground truth or trigger suppression). This truth-layering recurs in the infragraph confidence table, the lesson materializer (`resolves@1.0` vs regex-mined `affects@0.7`), and the chaos provenance label. The predecessor's trap: **provenance is reconstructed from a sidecar, not stored on the edge**, and the MAX-merge never rewrites `source`, so a NetBox edge later strengthened by LibreNMS misreports provenance. TG must give edges a first-class provenance column.

**VI. Evidence must be forward-only and falsifiable with a real negative control — never retrodiction, never self-scoring.** Commit the prediction at action time, verify against reality afterward; run a genuine degree-preserving shuffle and require the real prediction to be **≥2× as precise (`control_ratio ≤ 0.5`)** or the gate encodes nothing; seal the holdout and never tune to it; state *n* + the honest confidence ceiling next to any accuracy claim. Two hard-won calibrations: **raw structural blast-radius precision is ~0.05–0.15 BY DESIGN** (never gate on it — a PrecisionDrop alarm on it is permanently firing or permanently inert), and **real cascade confidence tops out ~0.70** ("no infra cascade is 95% deterministic"), which makes the aspirational 0.95-precision-@-0.8-confidence auto-resolve gate *structurally unsatisfiable* — only reversible fold/dedup at 0.80 tolerates the stochastic ceiling.

**VII. Substrate fragility silently defeats governance — the design survives the port, the engine must not.** `team_formation.py` emitted 0 rows because the n8n sandbox blocks `child_process`; host-local sentinel files are `rm`-able; the JSONL tail and `/tmp` acks are unshareable; `GATEWAY_DB` defaulted to the wrong path and split the telemetry. TG's deliberate subtractions — Temporal (durable pause/resume/signals) replacing sentinel files + `kill -INT`, one bundled gateway replacing the 3-plane split-brain, one Postgres with a DML-only role replacing multiple SQLite writers — are correct precisely because they delete the substrate a governance layer cannot police.

**VIII. Two decoupled graphs, one host identity — and the load-bearing safety mechanics hide in the details.** Keep the causal/dependency graph (typed rel_types + dynamics + traversal, blast radius) strictly separate from the retrieval/knowledge graph (incident/lesson/chaos nodes, RAG), but unify **host identity** via a single canonical node so incident history and blast-radius topology can finally traverse together. And preserve the battle-earned details that a clean-looking rewrite drops: common-cause **siblings at 0.6×** (the 2026-05-08 pattern — 4 VMs flap while the shared node stays silent), **stateful-target-territory dominance**, cross-site exclusion **only when both sites are known and differ**, the **reactive-vs-clean boot gate** (an OOM/panic reboot must never be learned as "scheduled"), and jailbreak input **scoping to untrusted narrative** (screening the system's own generated steps produced the 55-hit false-positive flood). Each dropped detail silently re-opens a specific past incident.
